# Ripster — аудит настроек + унификация путей (директива владельца 2026-06-20)

Две задачи: **(A)** унифицировать пути сохранения; **(B)** вырезать настройки,
не нужные публичным юзерам (репо с GitHub). Это исследование + план; исполнение —

## A. Унификация путей сохранения

### Цель (как сказал владелец)
Убрать per-engine «Путь сохранения» из настроек всех движков. Оставить **ОДИН общий
путь** в общих настройках. Внутри — папки по сервису и качеству:
```
<save-path>/<service>/<quality>
    …/beatport/320
    …/deezer/FLAC
    …/apple/ALAC (Lossless)
    …/tidal/FLAC CD
```

### ⚠️ ПОЧЕМУ ТОЛЬКО СОГЛАСОВАННО (важно)
Путь резолвится в ДВУХ местах:
1. `ripster/service_config.py::get_save_path` — для runner (disk-truth / манифест /
   подсчёт файлов на диске).
2. КАЖДЫЙ движок `build_cmd` сам читает `config.get("<svc>-save-path") or "save-path"`.

Если поменять только одно — манифест ищет в новом пути, движок качает в старый →
**«файлов нет» / фантомный успех** (тот самый класс багов проекта). Значит меняем
обе точки + UI **одним проходом**.

### План изменений (атомарно)
1. **`service_config.py`**:
   - `get_save_path(config, service, quality)` → `Path(<save-path>, <service>, <quality_folder>)`.
     Убрать `_SVC_PATH_KEYS` и Apple-спец-ветки (`alac/aac/atmos-save-folder`).
   - `_quality_folder_name` — оставить (даёт имя качества: `FLAC`, `320`, `ALAC (Lossless)`…).
   - `quality-subfolders` тумблер — упразднить (структура теперь всегда service/quality).
   - Обновить тесты `tests/test_service_config.py` под новую структуру.
2. **Движки** (`build_cmd`) — заменить чтение своего ключа на
   `from ripster.service_config import get_save_path; out = get_save_path(config, svc, quality)`:
   amazon, deezer, qobuz, tidal, orpheus_spotify, orpheus_beatport, soundcloud,
   sc_widevine, yandex, gamdl (10 шт). Так движок и runner всегда согласованы.
3. **UI `static/views/settings.html`** — удалить ~8 инпутов «Путь сохранения» из
   каждого сервиса; оставить ОДИН в блоке «📁 Пути сохранения» (общий `save-path`).
4. **Конфиг/дефолты** — убрать `*-save-path`, `atmos-path`, `aac-path`,
   `alac/aac/atmos-save-folder`, `quality-subfolders` из DEFAULT_CONFIG + миграция:
   при старте, если заданы старые ключи — игнор (один `save-path` главный).
5. **Снести** старые per-service пути читаются нигде → grep-проверка после.

Ключи на удаление: `qobuz-save-path, deezer-save-path, tidal-save-path,
soundcloud-save-path, beatport-save-path, orpheus-save-path, yandex-save-path,
amazon-save-path, zotify-save-path, spotiflac-save-path, atmos-save-folder,
aac-save-folder, alac-save-folder, atmos-path, aac-path, quality-subfolders`.

## B. Аудит настроек: что оставить публичным, что вырезать

Публичные вкладки: Очередь · Поиск · История · BBC · SC · Кодер · Спектр · Настройки
· Setup · Консоль. БЕЗ бота, гостей, Библиотеки, Теггера, Статистики.

### ✂️ ВЫРЕЗАТЬ из публичной версии (owner/internal)
| Блок настроек | Почему |
|---|---|
| 🤖 Конфигурация бота | бот — приватный аддон, вне публичной версии |
| 🔗 Гостевые ссылки | гостевой режим вырезан |
| `auto-delete-minutes` (ползунок хранения, settings.html:89) | владелец тестит; юзеру файлы удалять не нужно |
| ~~🐳 Docker Wrapper~~ → **НЕ вырезать, сделать ОПЦИОНАЛЬНЫМ** | Apple качается БЕЗ Docker через публичный wrapper (AMD/wm.wol.moe, без аккаунта) — это дефолт. Docker = опц. апгрейд для своих (Docker + премиум Apple-аккаунт = локальный lossless). Перенести в Setup как «продвинутое», дефолт = публичный wrapper. Нет Docker → Apple всё равно работает. |
| 🌍 Удалённый доступ (`public-url`, `remote-enabled`) | туннель владельца; публике локально не нужен (оставить выключенным/скрытым) |
| 🟢 Новые релизы / 📡 Радар релизов (Spotify release scanner) | фоновый сканер релизов владельца; риск бана аккаунта, не для публики |

## C. Блок «ℹ️ О сервисе» — обновление + поддержка автора (директива 2026-06-20)
В существующий блок «О сервисе» (settings.html) добавить:
1. **Обновление** — карточка апдейтера: текущая версия, кнопка «Проверить обновление»
   (`/api/update/check` → `latest`/`changelog`/`available`), и «Обновить» (`/api/update/apply`).
   Т.е. вся инфа об апдейте живёт здесь, в «О сервисе».
2. **Поддержка автора** — аккуратный блок БЕЗ имён: «Автор идеи и разработки» /
   «Сделано одним человеком» + кнопка/ссылка «Поддержать». Тон — тёплый, ненавязчивый.
   🔒 **HARD-CONSTRAINT (директива владельца): ПОЛНАЯ анонимность для всех — нигде
   ни имени, ни инициалов, ни телефона, ни номера карты.** Поэтому:
   - **ВЫБРАНО НАПРАВЛЕНИЕ: КРИПТА** (даёт настоящую анонимность — нет площадки-посредника,
     которая знает личность; получение на self-custody кошелёк без KYC).
   - **МОДЕЛЬ:** в блоке «Поддержать» — крипто-адрес(а) + QR + кнопка «Скопировать»,
     прямо в приложении (без внешней площадки). Юзер копирует адрес / сканирует QR → шлёт.
   - **Монеты/сети (рекоменд.):** USDT-TRC20 (TRON) — стейблкоин $1, копеечная комиссия,
     стандарт в РФ/СНГ; опц. TON (Telegram-нативный). Кошелёк: Trust Wallet / Tonkeeper.
   - **Анонимность:** получение полностью анонимно (адрес не привязан к имени, площадки нет).
     Кэш-аут в рубли — отдельный шаг со своими нюансами (P2P/обменники, иногда KYC) — но это
     не влияет на анонимность ПОЛУЧЕНИЯ и нигде в Ripster не светится.
   - **Карты (запас, не аноним к площадке):** Boosty/DonationAlerts под ником — если захочется
     принимать ещё и с карт. Telegram Stars / маркетплейсы — НЕ используем.
   - **Безопасность (критично):** seed-фраза (12–24 слова) = доступ ко ВСЕМ деньгам.
     Хранить офлайн, никому не показывать, никуда не вводить. Потерял — деньги пропали навсегда.
   - **Код:** конфиг-ключи `crypto-usdt-trc20`, `crypto-ton` (пусто → строка скрыта);
     адреса НЕ хардкодим в репо — владелец вписывает свои в Настройках.
   - **НЕЛЬЗЯ (палит имя/инициалы):** СБП по номеру телефона и перевод по номеру карты —
     банк показывает отправителю «Имя О.» получателя. Эти методы ИСКЛЮЧИТЬ.
   - Граница: анонимность для ЮЗЕРОВ/публики достижима; платформа выплат (Boosty/банк)
     по закону РФ знает личность (KYC) — но это приватно между тобой и платформой,
     нигде в Ripster/публике не светится.
   - В коде — конфиг-ключ `support-url` (пусто → блок скрыт), значение = ссылка на
     ник-страницу (boosty.to/ник). Личные реквизиты в репо НЕ коммитятся.
3. **Благодарности / Credits** — указать ассистента (по просьбе владельца), НЕ раскрывая
   человека: «Идея и проект — Автор» + «Разработка при участии Claude (Anthropic)».
   Это называет инструмент, а не личность → анонимность владельца сохраняется.

## D. Tidal-панель: «вход есть, но токен истёк» — это ДВА разных токена (не дубль)
Симптом: device-flow показывает «✓ вход (DE) / ✓ Tidal работает», но рядом «🔴 токен ИСТЁК».
Причина — **два пути токена**:
- **Скачивание** → device-flow TV-сессия (OrpheusDL loginstorage, само-рефреш) — РАБОТАЕТ. ✓
- **Поиск/метаданные (probe)** → старый ручной `tidal-token` из config (listen.tidal.com) —
  ИСТЁК → «🔴 токен истёк» + лог «[tidal] probe 401». ✗
Это не косметика: ручной токен ещё питает поиск/мету. **План:**
1. Probe/поиск перевести на свежий токен device-flow-сессии (`_orpheus_access_token()`),
   чтобы ручной токен стал НЕ нужен, а индикатор отражал рабочую сессию.
2. После этого блок «🔑 Токены (вручную)» + DevTools-инструкция + import-JSON свернуть
   в «Продвинутое» (или убрать) — device-flow рекомендован и само-рефрешится.
3. Индикатор «токен истёк» убрать/переключить на статус device-flow-сессии.
